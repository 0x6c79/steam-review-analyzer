# Steam Review Analysis Project

This project provides a comprehensive solution for collecting, analyzing, and visualizing Steam game reviews. It is designed to help users gain insights into player sentiment, identify key themes, and understand trends within review data. The project is structured into two primary components: a data scraper and a data analysis module.

## Project Overview

This project contains two main Python scripts:

### `steam_review_scraper.py`

This script is responsible for collecting raw review data from the Steam platform.

*   **Purpose:** Scrapes reviews for a specified Steam application ID.
*   **Key Features:**
    *   **Asynchronous HTTP Requests:** Leverages `aiohttp` and `asyncio` for efficient, non-blocking network requests, enabling concurrent data collection and significantly improving scraping speed.
    *   **Data Flattening:** Automatically flattens nested JSON structures (e.g., 'author' dictionary within reviews) into a single-level format for easier processing.
    *   **Timeout Mechanism:** Includes a global 3-minute timeout to prevent indefinite execution during scraping.
    *   **Rate Limiting & Error Handling:** Basic error handling for HTTP requests is implemented to catch `aiohttp.ClientError` exceptions, ensuring robustness.
*   **Output:** Saves the scraped and processed review data into a CSV file named `reviews.csv`.

### `analyze_reviews.py`

This script takes the `reviews.csv` output from the scraper and performs in-depth analysis and visualization.

*   **Purpose:** Provides insights into review data through sentiment analysis, keyword extraction, and various visualizations.
*   **Key Features:**
    *   **Sentiment Analysis:** Determines the emotional tone of reviews.
    *   **Keyword Extraction:** Identifies important terms and phrases within reviews.
    *   **Comprehensive Visualizations:** Generates a wide array of `.png` image files to visualize different aspects of the review data, including:
        *   Sentiment distribution (overall and over time)
        *   Keyword frequencies and word clouds (positive and negative, overall and by language)
        *   Sentiment breakdown by language, purchase type, playtime, early access status, and review length.
    *   **Performance Optimization:** Optimized for efficient data flow and reduced DataFrame copies during analysis.
*   **Dependencies:** Requires `nltk` (for stopwords) and `jieba` (for Chinese tokenization). It also handles the download of necessary `nltk` data and `langdetect` profiles. A custom font (`wqy-MicroHei.ttf`) is used for proper display of Chinese characters in plots.
*   **Output:** Produces numerous `.png` image files in the project root directory, each representing a different visualization of the review data.

# Building and Running

This project uses a Python virtual environment to manage its dependencies.

## Setup

1.  **Create a virtual environment:**
    ```bash
    python3 -m venv .venv
    ```
2.  **Activate the virtual environment:**
    ```bash
    source .venv/bin/activate
    ```
3.  **Install dependencies:**
    ```bash
    pip install -r requirements.txt
    ```

## Running the Project

The project can be run using the `main.py` entry point, which orchestrates the scraper and analyzer. Always activate the virtual environment first.

```bash
source .venv/bin/activate
```

### Running the Scraper Only

To run only the Steam review scraper, you must provide a Steam Application ID (`--app_id`). Optionally, you can limit the number of reviews fetched (`--limit`).

```bash
python3 main.py --scrape --app_id <YOUR_APP_ID> [--limit <NUMBER_OF_REVIEWS>]
```

### Running the Analyzer Only

To run only the review analyzer, ensure that `reviews.csv` (generated by the scraper) is present in the project root.

```bash
python3 main.py --analyze
```

### Running Both Scraper and Analyzer Sequentially

To run both the scraper and then the analyzer in sequence, use the `--all` flag. An `--app_id` is required.

```bash
python3 main.py --all --app_id <YOUR_APP_ID> [--limit <NUMBER_OF_REVIEWS>]
```

### Real-time Logging (for Scraper)

To view the scraper's output in real-time on the console while also saving it to a log file (`steam_review_scraper.log`), you would typically pipe the output. When running via `main.py`, the output will be directed to the console. If you need to log the scraper's output specifically, you would modify the `run_scraper` function in `main.py` to handle file logging, or redirect the `main.py` output itself:

```bash
python3 main.py --all --app_id <YOUR_APP_ID> | tee steam_review_scraper.log
```

# Review Analysis

The `analyze_reviews.py` script (now typically run via `main.py --analyze` or `main.py --all`) processes the `reviews.csv` file generated by the scraper to provide insights into the review data. It performs sentiment analysis, extracts keywords, and generates various visualizations. Performance has been optimized by reducing unnecessary DataFrame copies and ensuring efficient data flow between analysis modules.

*   `positive_wordcloud.png`: Word cloud for positive reviews (by language).
*   `negative_wordcloud.png`: Word cloud for negative reviews (by language).
*   `sentiment_by_language.png`: Sentiment distribution broken down by review language.
*   `sentiment_by_purchase_type.png`: Sentiment distribution based on whether the game was purchased on Steam.

# Development Conventions

*   **Asynchronous Operations:** The script leverages `asyncio` and `aiohttp` for efficient, non-blocking network requests.
*   **Error Handling:** Basic error handling for HTTP requests is implemented to catch `aiohttp.ClientError` exceptions.
*   **Data Flattening:** Nested JSON data, specifically the 'author' dictionary within each review, is flattened into a single level for easier consumption in the CSV output.
*   **Timeout Mechanism:** A global timeout of 3 minutes is implemented for the entire scraping process to prevent indefinite execution.
